# Architecture: Enhanced Markdown Support

## Strategy: Scoped Reset
To ensure consistent rendering of markdown elements across diverse host environments, we will use a scoped reset within the assistant message container.

## Component Structure

### 1. Parser Configuration (src/utils/chat-renderer.js)
The `marked` parser will be configured as a centralized singleton or with a standard configuration object:
- `gfm: true`
- `breaks: true`
- `headerIds: false` (to avoid conflicts)

### 2. Sanitization (src/utils/chat-renderer.js)
`dompurify` will remain the final gate, ensuring that any HTML generated by `marked` (including tables and lists) is safe.

### 3. Styling Logic (src/scss/_markdown.scss)
A new SCSS file will be created to house all markdown-specific styles. This file will be imported into `akvo-rag.scss`.

#### Nested Selector Pattern:
```scss
.akvo-msg-assistant {
  ul, ol {
    // Explicit reset and styling
    list-style: disc;
    margin: 10px 0 10px 20px;
    padding: 0;
  }

  table {
    width: 100%;
    border-collapse: collapse;
    // ...
  }
}
```

## Data Flow
1. **Input**: Cleaned streaming text.
2. **Transform**: `marked.parse(text, config)` -> Raw HTML.
3. **Purify**: `DOMPurify.sanitize(html)` -> Safe HTML.
4. **Render**: Safe HTML injected into `.akvo-msg-assistant`.
5. **Style**: CSS rules in `_markdown.scss` apply based on tag types.
